# themes

Canonical theme definitions (source of truth).

Conventions:
- One folder per theme.
- Split into `light.yml` and `dark.yml` when both exist.
- YAML style: 2-space indent, quote hex colors, avoid anchors/aliases.
- Prefer documenting:
  - palette token naming
  - semantic roles
  - contrast constraints

See `docs/theme-format.md` for the recommended format and style rules.
